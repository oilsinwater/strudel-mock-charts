QUALITY BENCHMARK TASKFLOW - ASCII WIREFRAMES
==============================================

WIREFLOW DIAGRAM:
-----------------

[INDEX] --applyFilter--> [INDEX]                    // filter assessments
[INDEX] --navigateToAssess(id)--> [ASSESS]          // select dataset to assess
[ASSESS] --runQualityChecks--> [RESULTS]            // run checks and show results
[INDEX] --navigateToTrends--> [TRENDS]              // view quality trends
[RESULTS] --navigateToAssess--> [ASSESS]            // re-assess dataset

COMPONENT MAPPING:
------------------

- PageHeader: Standard STRUDEL page header component
- Filters: FilterPanel with quality metric filters
- AssessmentGrid: DataGrid showing assessment history
- QualityPreview: Preview panel for selected assessment
- PrimaryActions: Action buttons (Assess New, Trends)
- DatasetInfo: Dataset metadata and connection info
- QualityChecks: Configurable quality check options
- ValidationResults: Real-time validation results display
- AssessmentActions: Assessment workflow controls
- QualityMetrics: Quality score breakdown display
- ValidationDetails: Detailed validation results
- TrendCharts: Quality trend visualization
- ResultActions: Export, re-assess, navigation actions
- TrendFilters: Date range and dataset filters
- QualityTrendCharts: Multi-dataset trend comparison
- DatasetComparison: Side-by-side dataset quality comparison
- TrendActions: Trend analysis export and navigation

CONNECTION TO EXPLORE-DATA:
---------------------------

The quality-benchmark taskflow connects to explore-data through:

1. URL Parameters: ?ids=ds-001,ds-002&origin=explore-data
2. Shared Dataset IDs: Both taskflows reference same dataset identifiers
3. Cross-navigation: "Benchmark Quality" button in explore-data
4. State Hydration: Pre-select datasets from explore-data context
5. Filter Mapping: Convert explore-data filters to quality metrics

User Journey:
explore-data (browse datasets) → quality-benchmark (assess quality) → detailed reports
